# 🎯 Plano de Estudos: Backend Moderno (30 Dias)

> [!NOTE]
> Este plano de estudos é o **primeiro curso** da minha jornada no **[Bootcamp Santander 2026 - AI Java Back-end](../README.md)**.

Bem-vindo ao seu espaço de estudos! Este repositório foi estruturado para materializar o plano de 30 dias desenvolvido para consolidar sua compreensão dos pilares do **Backend Moderno**. 

Aqui você acompanhará seu progresso, fará anotações diárias, organizará seus materiais de referência e executará projetos práticos para dominar o desenvolvimento de APIs RESTful, containerização, microsserviços, mensageria e deploy em nuvem.

---

## 📌 Objetivo do Plano
Ao final de 30 dias, você será capaz de:
* 🛠️ **Projetar e implementar** APIs RESTful robustas.
* 🐳 **Containerizar aplicações** utilizando Docker e Docker Compose.
* 🗺️ **Compreender a arquitetura de microsserviços** e seus padrões comuns.
* ✉️ **Integrar sistemas assíncronos** usando mensageria (Message Brokers).
* ☁️ **Realizar deploy** de aplicações backend na nuvem com pipelines de CI/CD.

## 💡 Como Utilizar Este Espaço

1. **Siga o Roteiro**: Acesse o [`cronograma.md`](cronograma.md) para saber exatamente quais conceitos estudar e quais práticas realizar a cada bloco de dias.
2. **Registre seu Progresso**:
   * Crie uma cópia do arquivo [`anotacoes/semana-01.md`](anotacoes/semana-01.md) para as semanas seguintes (ex: `anotacoes/semana-02.md`).
   * Use esses arquivos para documentar o que aprendeu, registrar códigos úteis, anotar dúvidas para pesquisar depois e marcar as tarefas concluídas.
3. **Organize seus Recursos**: No arquivo [`recursos.md`](recursos.md), centralize links de tutoriais, artigos e cursos que encontrar ou que já possuir em seu e-mail/Drive.
4. **Git é seu amigo**: Faça commits diários das suas práticas e anotações. Isso criará um histórico rico de evolução e construirá um excelente portfólio no GitHub!

---

## ⚡ Dicas Finais para o Sucesso

* **Não decore, compreenda**: O objetivo principal é entender os *porquês* (por que usamos Docker? Qual problema a mensageria resolve?) e não apenas memorizar comandos ou sintaxes.
* **Mão na massa**: Dedique tempo de qualidade para as práticas sugeridas ao final de cada bloco de dias. A teoria se consolida quando digitamos código e resolvemos erros.
* **Itere no seu ritmo**: Se precisar de mais tempo em algum conceito complexo, ajuste o cronograma. O importante é a qualidade do aprendizado!
